Special instructions

Steps to stop the service, get the remaining files from the DM work file, and manually concatenate the error file.

  1. Stop the TSA.SERVICE record for the specific DM service. Under the running services, using COS TSS.LIST, find the ID of the running DM service.
  2. Stop the DM.SERVICE.CONTROL DSC of the running service.
  3. Get the unprocessed pending records from the DM service work file, F.DM.SERVICE.DATA.FILE. Directly from the database, using SSMS or DbBeaver, run: 
    SELECT * FROM F_DM_SERVICE_DATA_F000
  4. Export the result to CSV and share it with Aspire.
  5. Concatenate the error files manually. In the directory where the file is placed, run: 
    cat APC.R23.FTNOS#LOG.* > APC.R23.FTNOS-ERROR.YYMMDD.txt
  6. Clean up the F.DM.SERVICE.DATA.FILE and the JOB.LIST related to the service.
    1. Run JQL CLEAR-FILE F.DM.SERVICE.DATA.FILE after steps 3 and 4 are completed.
    2. Run JQL LIST-ITEM F.LOCKING WITH @ID LIKE ...JOB....
    3. Choose the Job List associated with the DM service. Confirm by running LIST-ITEM F.JOB.LIST.x.
    4. Run JQL CLEAR-FILE F.JOB.LIST.x.
    5. Run JED F.LOCKING F.JOB.LIST.x, then press d and then Y.
